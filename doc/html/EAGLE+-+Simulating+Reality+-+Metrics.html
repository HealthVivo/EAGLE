Introduction

We follow here the evolution of EAGLE based on metrics.

EAGLE Versions

  • Real1: Real dataset
  • v1.0: No homopolymer model, Qtable without dependencies
  • v20120307: Qtable with dependencies
  • v20120321: With homopolymer model

Each of these versions is able to generate 3 BAM files (except the real datasets, which only have the first 2 files):

  • CASAVA sorted.bam
  • sorted.bam + sorted.realigned.bam
  • sim.bam, which contains exactly what the simulator simulated

Each metric is ran for each of these BAM files, in order to show the effect of the aligner.

Metrics

  • Metric M1: Histogram of the number of mismatches per read
    Metric M1a: quantile 99.999%
    Metric M1b: smallest X(=number of mismatches) where Y(=number of X)<sum(values)/100000
  • Metric M2: Error rate
    Metric M2a: Mismatch rate
    Metric M2b: Insertion rate
    Metric M2c: deletion rate
  • Metric M3: How mismatch rate changes across the genome: standard deviation of mismatch rate for genome windows of size 10kbases, ignoring areas of low coverage.
  • Metric M4: How coverage changes across the genome: standard deviation of coverage for genome windows of size 10kbases, ignoring areas of low coverage.
