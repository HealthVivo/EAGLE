Software development : EAGLE Use Case - Simulating a small Run Folder from PhiX

This page last changed on Oct 07, 2013 by ljanin.

Preamble

If you haven't done it yet, you may refer to the EAGLE - Installation and configuration instructions.
We are using EAGLE to simulate a small Run Folder from PhiX by going through the following steps:

  1. Sample simulation, by applying known mutations to a PhiX reference genome
  2. Sequencing simulation with 1 tile in 1 lane

Input and configuration files

Actually, instead of PhiX we decided to use a lovely Enterobacteriophage.

In this example we will use the following inputs:

  • Our Enterobacteriophage reference genome
  • RunInfo_PairedReads1x1Tiles.xml
  • Variants/OneOfEachType.vcf

Initialisation

source /illumina/development/EAGLE/stable-setup

configureEAGLE.pl \
  --run-info=$EAGLE_SHARE/RunInfo/RunInfo_PairedReads1x1Tiles.xml \
  --reference-genome=/illumina/scratch/iGenomes/Enterobacteriophage_lambda/NCBI/1993-04-28/Sequence/Chromosomes/ \
  --variant-list=$EAGLE_SHARE/Variants/OneOfEachType.vcf \
  --coverage-depth=30

Execution

cd EAGLE
make

This should take about 10 seconds to complete and generate a Run Folder full of BCL files, ready to be aligned by your preferred aligner tool.

You can also generate BAM files (for visualisation in IGV for example):

make bam

Results

The main result is the generated Run Folder.
Based on the chromosome length (48502 bases), the read length (202 bases) and the desired coverage (30x), the simulator generated 7203 paired reads.

You can inspect the Run Folder by hand.
